# ramdb_db_fnc_hashRemove

## Description
Removes a specific field from the hash table associated with the current/specific client/player. This function deletes a single field and its associated value without affecting other fields in the client's hash table.

## Syntax
```sqf
[_key, _keyField] call ramdb_db_fnc_hashRemove
```

## Parameters
| Parameter   | Type   | Description                         | Default |
|-------------|--------|-------------------------------------|---------|
| `_key`      | String  | Key to remove the field from       | ""      |
| `_keyField` | String | Name of the field to be removed     | ""      |

## Return Value
None.

## Examples
### Remove a specific field:
```sqf
["", "loadout"] call ramdb_db_fnc_hashRemove;
[getPlayerUID player, "loadout"] call ramdb_db_fnc_hashRemove;
```

### Remove a field remotely:
```sqf
["playerSettings"] remoteExecCall ["ramdb_db_fnc_hashRemove", 2, false];
```

## Notes
- Only removes a single field from the current/specific client's hash table
- Validates that the key field parameter is not empty before proceeding
- Does not affect other fields in the hash table
- If the field doesn't exist, the operation has no effect
- The operation is executed immediately and synchronously
- All operations are logged for debugging purposes

## Related Functions
- `ramdb_db_fnc_hget`: Retrieves a field value from the current/specific client's hash table
- `ramdb_db_fnc_hgetall`: Retrieves all fields from the current/specific client's hash table
- `ramdb_db_fnc_hset`: Sets a field value pair in the current/specific client's hash table
- `ramdb_db_fnc_hmset`: Sets multiple field value pairs in the current/specific client's hash table
