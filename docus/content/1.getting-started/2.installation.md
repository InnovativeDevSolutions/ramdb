# Installation

This guide will walk you through installing and configuring ArmaRAMDb for your Arma 3 server or local game.

## Requirements

- Arma 3 (Latest version recommended)
- [CBA_A3](https://steamcommunity.com/sharedfiles/filedetails/?id=450814997) (Community Base Addons)
- Windows or Linux operating system

## Step 1: Download ArmaRAMDb

Download the latest version of ArmaRAMDb from one of these sources:

::callout{icon="i-simple-icons-github" color="secondary"}
[Download from GitHub Releases](https://github.com/InnovativeDevSolutions/ramdb/releases/latest)
::

::callout{icon="i-simple-icons-steam" color="primary"}
Subscribe on [Steam Workshop](https://steamcommunity.com/sharedfiles/filedetails/?id=3646149174)
::

## Step 2: Create @ramdb Folder

Create a folder named `@ramdb` in your Arma 3 root directory. This is the same location as:
- `arma3_x64.exe` (for client)

## Step 3: Copy config.xml

Copy the `config.xml` file from the mod folder into `@ramdb\config.xml`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<RamDbConfig>
    <ContextLog>false</ContextLog>
    <Debug>false</Debug>
    <AutoBackup>true</AutoBackup>
    <BackupFrequency>15</BackupFrequency>
    <MaxBackups>10</MaxBackups>
</RamDbConfig>
```

### Configuration Options

| Setting | Type | Description | Default |
|---------|------|-------------|---------|
| `ContextLog` | Boolean | Log call context information (Steam ID, file source, etc.) | `false` |
| `Debug` | Boolean | Enable debug logging | `false` |
| `AutoBackup` | Boolean | Enable automatic backups | `true` |
| `BackupFrequency` | Integer | Minutes between automatic backups | `15` |
| `MaxBackups` | Integer | Maximum number of backups to keep | `10` |

::callout{icon="i-lucide-alert-triangle" color="warning"}
**Note**: Setting `Debug` to `true` will generate large log files. Only enable for troubleshooting.
::

## Step 4: Load the Mod

### For Server:

Add the mod to your server startup parameters:

```bash
-mod=@CBA_A3;@ramdb
```

**Example startup line:**
```bash
arma3server_x64.exe -mod=@CBA_A3 -serverMod=@ramdb -config=server.cfg -port=2302
```

### For Client:

1. Launch Arma 3
2. Go to **Launcher** â†’ **Mods**
3. Enable **@CBA_A3** and **@ramdb**
4. Click **Play**

## Step 5: Verify Installation

Once Arma 3 launches, check your RPT log file to verify ArmaRAMDb loaded correctly:

**Location of RPT files:**
- **Client**: `C:\Users\[YourUsername]\AppData\Local\Arma 3\`
- **Server**: Usually in the same directory as the server executable

Look for these lines:

```
ArmaRAMDb: DLL Version 1.0.0 found
ArmaRAMDb: Functions loaded and Initialization completed!
ArmaRAMDb: Buffer size set to 20480 Bytes
ArmaRAMDb: Ready for use!
```

::callout{icon="i-lucide-check-circle" color="primary"}
**Success!** If you see these messages, ArmaRAMDb is installed and ready to use.
::

### Config File Not Found

If you see "Config file not found! Default settings will be used" in the logs:

1. Verify `config.xml` exists in `@ramdb\config.xml`
2. Check the file is not `config.xml.txt` (Windows may hide extensions)
3. Ensure the XML structure matches the example above

### Permission Issues (Linux)

On Linux servers, ensure the extension file has execute permissions:

```bash
chmod +x @ramdb/ArmaRAMDb_x64.so
```
